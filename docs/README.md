# Docs Guide

This folder is for two things:

1. explain how the project works
2. capture why decisions were made so you can study them later

## Suggested structure

- `README.md`: index for the docs folder
- `architecture.md`: how the system fits together
- `datasets.md`: where training data comes from now and what to use later
- `implementation-log.md`: what was built, in order, and why
- `research-roadmap.md`: what to learn next and what to upgrade next

When the project grows, add folders like:

- `experiments/`: one markdown file per meaningful training run
- `decisions/`: short design records when we choose between approaches
- `papers/`: notes from research papers in your own words

## How to use this folder well

- Keep explanations short and concrete.
- Write down tradeoffs, not just outcomes.
- Prefer "we chose X because Y" over vague summaries.
- When code changes, update the matching doc on the same day.
